# Agent Sprint Template (v7.6)

## Sprint Structure
- **S0**: baseline/contracts
- **S1**: state contract repair
- **S2**: diagnostics hardening
- **S3**: testing/lanes
- **S4**: repo hygiene
- **S5**: docs refresh
- **S6+**: reinforcement/feature tracks (A/B/C when authorized)

## Sprint Overview
- Goals and scope for the sprint.
- Patchset list with owners and lanes required.
- Risk register (what can break, mitigation).
- Test impact plan (fast vs runtime).

## Patchset Roadmap
- For each patchset: title, objective, files, risks, required tests, acceptance criteria.
- Dependencies/order between patchsets.
- Handoff notes between agents.

## Continuity
- Keep a running log of applied patchsets and remaining tasks.
- Record skipped tests and why.
- Update manifest/docs as patchsets land.
