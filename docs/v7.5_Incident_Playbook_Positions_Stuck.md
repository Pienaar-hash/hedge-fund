# v7.5 Incident Playbook — Positions Stuck

Symptoms
- Open positions stay unchanged for hours; no new trades fire.
- TP/SL not triggering; PnL appears stale or flat.
- Dashboard shows positions but no recent execution/exit activity.

Checklist
1) Risk snapshot — `logs/state/risk_snapshot.json`
   - Check `risk_mode`, `dd_state`, circuit breaker flags.
   - Confirm NAV freshness and drawdown values.
2) Positions state — `logs/state/positions_state.json`
   - Ensure open positions have non-zero `entry_price` / `mark_price`.
   - Verify quantities and symbols are correct.
3) Ledger / TP/SL registry
   - `positions_ledger.json` and `position_tp_sl.json`
   - Compare counts: open positions vs TP/SL registrations.
4) Runtime diagnostics — `logs/state/diagnostics.json`
   - `veto_counters.by_reason` to see dominant veto gate.
   - Exit pipeline: `tp_sl_missing_count`, `underwater_without_tp_sl_count`, timestamps of last scan/trigger.
5) Recovery options
   - If vetoed by caps (`max_concurrent`, `symbol_cap`, `portfolio_dd_circuit`), adjust config only if intentional.
   - If TP/SL missing, run `scripts/seed_exit_registry_from_open_positions.py` or rebuild ledger.
   - If exit scans stale, restart executor and confirm `diagnostics.json` timestamps update.
